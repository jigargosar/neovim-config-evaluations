# CLAUDE.md

Static orientation for this repo. Answers **what** this is and **why**.
For **where we are / where to go next**, read `docs/STATUS.md`.

## What this is

This repo supports two tracks:

- **Track 1 — config evaluation (parent goal).** Evaluate two Neovim configs and
  settle on one daily driver. See `docs/roadmap.md`.
- **Track 2 — the learning tool (active work).** Build a tutorial to learn all
  MiniMax plugins (~35 MINI modules + 5 external).

## Why

MiniMax is a large config. The fastest way to actually learn it is a tutorial
that, per plugin, shows a real demo, points at the official `:h`, and tests
recall — rather than reading config files cold.

## The learning tool — design

A **data-driven "book"**. Each chapter = a real demo **visual** + `:h`
cross-references + a quiz + a cheatsheet. Visuals are generated from real Neovim
with **VHS** (a `.tape` script drives nvim and renders to GIF/MP4).

Why a book and not an interactive emulator: emulating each plugin's behavior does
not scale to ~40 modules. A book with real recorded demos does. Full rationale and
rejected approaches: `docs/meta-tutorial-book.md`.

## Structure

- `CLAUDE.md` — this file: static what + why.
- `docs/STATUS.md` — **living** current state + next steps (read this to resume).
- `docs/meta-tutorial-book.md` — design rationale; approaches tried and rejected.
- `docs/roadmap.md` — Track 1 (config evaluation) goal and notes.
- `docs/reference/` — `cheatsheet.md` and a clone of the MiniMax config
  (`minimax/configs/nvim-0.13/plugin/30_mini.lua` is the per-module source material).
- `docs/archive/half-hearted-simulation/` — the abandoned React simulator app,
  kept for reference (reusable quiz component lives here).
- `tools/vhs/` — VHS scripts plus their generated artifacts (e.g. `hello.gif`).
  This is an **experimental** workspace — we are still trying things here, so the
  artifacts are kept in-tree as proof/reference rather than treated as final output.

## Working style

- The repo is now **docs + tools** — there is no active application (the React app
  was archived). Do not re-add a build/workspace without a reason.
- Demo visuals are generated by running the `tools/vhs/` scripts inside **WSL2**
  (Ubuntu), writing output under the repo. Note: in WSL, `/mnt/c/...` is the same
  as Windows `C:\...`.
- General coding conventions (pnpm, TypeScript strictness, output style) come from
  the user's global `~/.claude` config, not this file.
